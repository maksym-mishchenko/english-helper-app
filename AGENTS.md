# AGENTS.md

## Coding Standards

All contributors and coding agents must follow the org-wide coding standards:
📖 **[CODING_STANDARDS.md](https://github.com/maksym-mishchenko/docs/blob/main/docs/CODING_STANDARDS.md)**

Key rules: SRP · DRY · KISS · 400-line file cap · 40-line function cap · tests required · conventional commits.

## Agent Rules
- Read CODING_STANDARDS.md before writing any code
- Use `git worktree` for parallel agent work — never two agents in the same worktree
- Conventional commits with descriptive messages
- Run tests before marking a task done
- Check existing patterns with grep before writing new code
